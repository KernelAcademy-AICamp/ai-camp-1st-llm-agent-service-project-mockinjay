# CareGuide/CarePlus - 종합 UX 테스팅 리포트
**테스트 날짜:** 2025-11-27
**테스터:** UX Persona Testing Panel
**애플리케이션:** CareGuide (신장병 환자 지원 AI 서비스)
**환경:** Development (http://localhost:5181)

---

## 📋 Executive Summary

CarePlus는 신장병 환자, 간병인, 연구자를 위한 특화 AI 서비스입니다. 전반적으로 깔끔한 디자인과 명확한 정보 계층 구조를 보여주지만, **백엔드 서버 미연결** 상태로 인해 대부분의 기능을 완전히 테스트할 수 없었습니다. 프론트엔드 UI/UX 관점에서의 평가를 진행했습니다.

**주요 발견사항:**
- ✅ **우수한 점:** 깔끔한 디자인, 명확한 네비게이션, 좋은 에러 처리
- ⚠️ **개선 필요:** 접근성 향상, 모바일 최적화, 텍스트 가독성
- ❌ **치명적 문제:** 백엔드 연결 실패로 핵심 기능 미작동

---

## 🎯 기능 테스트 결과

### ✅ 정상 작동하는 요소

| 기능 | 상태 | 세부사항 |
|------|------|---------|
| 페이지 네비게이션 | ✅ 정상 | 모든 링크가 올바른 페이지로 이동 |
| 랜딩 페이지 | ✅ 정상 | 3개 카테고리 버튼 작동 (의료복지, 식이영양, 연구논문) |
| 로그인 페이지 | ✅ 정상 | 폼 입력, 회원가입 링크 작동 |
| 회원가입 페이지 | ✅ 정상 | 3단계 진행 표시, 폼 구조 명확 |
| 퀴즈 페이지 | ✅ 정상 | 퀴즈 목록 표시, 난이도 선택 UI |
| 트렌드 페이지 | ✅ 정상 | 검색 UI, 안내 메시지 표시 |
| 사이드바 네비게이션 | ✅ 정상 | 6개 메뉴 항목 작동 (AI챗봇, 식단케어, 퀴즈미션, 커뮤니티, 트렌드, 대시보드) |
| 모바일 네비게이션 | ✅ 정상 | 하단 탭 바 작동 |
| 에러 핸들링 | ✅ 우수 | 백엔드 오류 시 명확한 메시지 표시 |

### ⚠️ 주의가 필요한 요소

| 기능 | 문제점 | 영향도 |
|------|--------|--------|
| 채팅 입력 | 빈 입력 시 전송 버튼 비활성화 (정상이지만 피드백 부족) | 낮음 |
| 프로필 선택 드롭다운 | 변경 시 UI 피드백 없음 | 중간 |
| 대화 목록 | 중복된 대화방 표시 (데이터 이슈일 가능성) | 중간 |
| 커뮤니티 "다시 시도" 버튼 | 재시도 시 로딩 상태 표시 없음 | 낮음 |

### ❌ 오류 발견 (백엔드 연결 실패)

| 기능 | 에러 메시지 | 예상 원인 |
|------|------------|----------|
| 채팅 메시지 전송 | `Failed to fetch` - http://localhost:8000/api/chat/stream | 백엔드 서버 미실행 |
| 로그인 | `Failed to fetch` - http://localhost:8000/api/auth/login | 백엔드 서버 미실행 |
| 커뮤니티 게시글 | `Failed to load resource` - /api/community/posts | 백엔드 서버 미실행 |
| 퀴즈 시작 | `Failed to fetch` - /api/quiz/session/start | 백엔드 서버 미실행 |

**참고:** 이는 개발 환경 설정 문제이며, 프론트엔드 코드 자체의 문제는 아닙니다.

## 💬 페르소나 토론 및 분석

### 페르소나 1: 🧓 할머니 영숙 (68세, 디지털 초보자)

**테스트 시나리오:** 처음 서비스를 방문하여 신장병 정보를 찾으려 함

**발견 사항:**

**👍 잘된 점:**
- 랜딩 페이지가 매우 단순하고 명확함
- 3개의 큰 버튼 (의료복지, 식이영양, 연구논문)으로 선택이 쉬움
- 아이콘과 텍스트가 함께 있어 이해하기 쉬움

**👎 문제점:**
1. **텍스트 크기가 작음** - 설명 문구가 읽기 어려움
2. **입력창 placeholder 텍스트가 희미함** - "궁금한 점을 물어보세요" 글자가 잘 안 보임
3. **"맞춤 정보: 환자(신장병 환우)" 드롭다운** - 이게 뭔지, 왜 바꿔야 하는지 설명이 없음
4. **채팅 인터페이스가 복잡함** - 사이드바, 탭, 옵션이 너무 많아 혼란스러움

**영숙 할머니의 코멘트:**
> "첫 화면은 좋은데, 채팅방 들어가니까 버튼이 너무 많아서 어디를 눌러야 할지 모르겠어요. 그리고 글씨가 작아서 돋보기 없이는 못 읽겠어요."

**우선순위 개선사항:**
- 🔴 **High**: 최소 16px 이상으로 텍스트 크기 증가
- 🔴 **High**: 드롭다운 옆에 물음표 아이콘 + 툴팁 설명 추가
- 🟡 **Medium**: 채팅 인터페이스 단순화 (처음 사용자는 Auto 모드만 보이기)

### 페르소나 2: 👨‍💻 개발자 민수 (32세, 파워유저)

**테스트 시나리오:** 빠르게 논문 검색과 트렌드 분석을 하려 함

**발견 사항:**

**👍 잘된 점:**
- 사이드바 네비게이션이 명확하고 빠름
- 키보드로 입력창 접근 가능
- 에러 메시지가 개발자 친화적 (URL 포함, 원인 설명)

**👎 문제점:**
1. **키보드 단축키 없음** - Ctrl/Cmd + K로 검색, ESC로 닫기 등이 없음
2. **로딩 상태 표시 부족** - 버튼 클릭 시 즉각적인 피드백 없음
3. **검색 자동완성 없음** - 트렌드 페이지에서 키워드 제안 없음
4. **대화 내역 검색 기능 없음** - "대화 검색..." 입력창이 있지만 작동 여부 불확실

**민수의 코멘트:**
> "디자인은 깔끔한데, 키보드만으로 작업하기 어렵네요. 그리고 채팅창에서 이전 대화를 검색하거나 북마크하는 기능이 있으면 좋겠어요. 로딩 시간이 긴 건지 버그인지 알 수가 없어요."

**우선순위 개선사항:**
- 🟡 **Medium**: 키보드 단축키 추가 (모달 닫기, 검색 포커스)
- 🟡 **Medium**: 버튼 클릭 시 로딩 스피너 표시
- 🟢 **Low**: 검색 히스토리 및 자동완성 기능

### 페르소나 3: 👩‍🎓 대학생 지은 (22세, 모바일 네이티브)

**테스트 시나리오:** 모바일에서 커뮤니티 글 읽고 퀴즈 풀기

**발견 사항:**

**👍 잘된 점:**
- 모바일 하단 네비게이션이 잘 작동함
- 반응형 디자인이 대체로 양호함
- 터치 타겟 크기가 적절함

**👎 문제점:**
1. **데스크톱 느낌이 강함** - 모바일 특화 인터랙션 부족
2. **스와이프 제스처 미지원** - 대화방 삭제, 뒤로가기 등
3. **풀 스크린 모드 없음** - 채팅 시 키보드가 UI를 가림
4. **다크모드 없음** - 밤에 사용하기 눈부심
5. **하단 네비게이션 레이블** - "더보기"가 모호함 (무엇의 더보기?)

**지은의 코멘트:**
> "앱처럼 느껴지지 않고 웹사이트를 모바일로 축소한 느낌이에요. 인스타처럼 스와이프해서 글을 넘기거나 대화방을 삭제할 수 없고, 다크모드도 없어서 밤에 쓰기 불편할 것 같아요."

**우선순위 개선사항:**
- 🔴 **High**: 다크모드 추가 (시스템 설정 자동 감지)
- 🟡 **Medium**: 스와이프 제스처 구현 (대화방 삭제, 뒤로가기)
- 🟡 **Medium**: 채팅 시 키보드 대응 개선 (입력창 위로 올리기)
- 🟢 **Low**: "더보기" → "설정" 또는 "메뉴"로 명확하게 변경

### 페르소나 4: 👨‍👩‍👧 워킹맘 수진 (38세, 시간 부족)

**테스트 시나리오:** 퀴즈를 시작했다가 중단하고 나중에 이어서 하려 함

**발견 사항:**

**👍 잘된 점:**
- 퀴즈 진행도 표시 (1/5 완료 20%)
- 학습 레벨과 포인트 시각화
- 완료한 퀴즈에 "완료" 배지 표시

**👎 문제점:**
1. **진행 중 저장 기능 불명확** - 퀴즈를 중간에 닫으면 저장되는지 모름
2. **북마크 기능 없음** - 나중에 다시 보고 싶은 채팅이나 논문 저장 불가
3. **알림 기능 없음** - 일일 퀴즈 알림, 새 게시글 알림 등
4. **빠른 작업 단축키 없음** - 자주 묻는 질문 템플릿 없음

**수진의 코멘트:**
> "퀴즈를 5문제 다 못 풀면 어떻게 되는지 모르겠어요. 저장이 되나요? 그리고 나중에 다시 보려고 중요한 대화를 저장하고 싶은데 북마크 기능이 없네요. 바쁜 엄마 입장에서 빠르게 자주 묻는 질문만 모아볼 수 있으면 좋겠어요."

**우선순위 개선사항:**
- 🔴 **High**: 진행 중 퀴즈 자동 저장 및 "이어하기" 기능
- 🟡 **Medium**: 채팅/논문 북마크 기능 (별표 아이콘)
- 🟡 **Medium**: FAQ 빠른 링크 추가 (랜딩 페이지에)
- 🟢 **Low**: 브라우저 알림 기능 (옵트인)

### 페르소나 5: 👨‍🦯 시각장애인 준호 (45세, 스크린리더 사용)

**테스트 시나리오:** 스크린리더로 채팅 기능 사용하기

**발견 사항:**

**👍 잘된 점:**
- 대부분의 버튼에 aria-label 또는 텍스트가 있음
- 헤딩 구조가 논리적임 (h1, h2, h3)
- 링크가 명확함 (예: "로그인", "회원가입")

**👎 문제점:**
1. **아이콘 버튼 alt text 부족** - 많은 아이콘 버튼이 "button"으로만 읽힘
   - 예: 전송 버튼 (종이비행기 아이콘), 리셋 버튼 등
2. **프로필 선택 드롭다운 레이블 불명확** - "Select user profile"은 영어
3. **에러 메시지 읽기 어려움** - 마크다운 포맷(**볼드**, URL)이 스크린리더에서 복잡함
4. **포커스 순서 문제** - 사이드바 토글 버튼이 논리적 순서와 맞지 않음
5. **이미지 alt text 없음** - 로고, 아이콘 이미지에 설명 없음

**준호의 코멘트:**
> "기본 구조는 괜찮은데, 버튼을 눌러봐야 뭘 하는 건지 알 수 있어요. 특히 아이콘만 있는 버튼들이 문제예요. 그리고 채팅 메시지를 읽을 때 볼드나 링크가 섞여 있으면 계속 끊겨서 읽혀요."

**우선순위 개선사항:**
- 🔴 **High**: 모든 아이콘 버튼에 aria-label 추가
  - "Send message", "Reset current session", "Close sidebar" 등
- 🔴 **High**: 이미지 alt 속성 추가 (로고: "CarePlus logo", 아이콘: 기능 설명)
- 🟡 **Medium**: 에러 메시지를 plain text로 간소화 (또는 aria-live로 읽기)
- 🟡 **Medium**: 포커스 순서 검토 및 수정 (tabindex)
- 🟢 **Low**: 키보드 포커스 표시 강화 (outline 스타일)

### 페르소나 6: 🌍 외국인 톰 (29세, 한국어 학습 중)

**테스트 시나리오:** 한국어가 서툴지만 아이콘과 UI로 서비스 이해하기

**발견 사항:**

**👍 잘된 점:**
- 아이콘 사용이 일관적임
- 색상 코딩이 명확함 (청록색 = 주요 액션)
- 에러 메시지가 친절함 (이모지 사용)

**👎 문제점:**
1. **다국어 지원 없음** - 영어, 중국어 등 선택 불가
2. **일부 아이콘이 모호함** - "트렌드" 아이콘 (📈)은 명확하지만, "식단케어" (🍽️)는 애매함
3. **전문 용어가 많음** - "만성콩팥병", "고인산혈증" 등 번역 어려움
4. **placeholder가 너무 구체적임** - "만성콩팥병에 대해 무엇이든 물어보세요" → 다른 질문 못할 것 같음

**톰의 코멘트:**
> "아이콘 덕분에 어디가 뭔지 대충은 알겠는데, 한국어를 못 읽으면 사용이 불가능해요. 영어 옵션이 있으면 좋겠고, 의학 용어는 너무 어려워요. 간단한 단어로 설명해주는 툴팁이 있으면 좋겠어요."

**우선순위 개선사항:**
- 🟡 **Medium**: 다국어 지원 (최소 한국어/영어)
- 🟡 **Medium**: 의학 용어 옆에 간단한 설명 툴팁
- 🟢 **Low**: 아이콘 일관성 검토 (Material Design 또는 공통 아이콘셋 사용)
- 🟢 **Low**: placeholder 텍스트 더 포괄적으로 변경 ("무엇이든 물어보세요")

## 🤝 페르소나 간 합의점

모든 페르소나가 동의한 **공통 개선사항:**

1. **텍스트 가독성 향상** (영숙, 준호, 톰 동의)
   - 최소 폰트 크기 16px
   - 명확한 대비 (WCAG AA 기준)
   - 행간 1.5 이상

2. **에러 처리 개선** (민수, 수진, 준호 동의)
   - 로딩 상태 명확히 표시
   - 재시도 버튼에 시각적 피드백
   - 평문 에러 메시지 (마크다운 지양)

3. **접근성 강화** (영숙, 준호, 톰 동의)
   - 모든 인터랙티브 요소에 레이블
   - 키보드 네비게이션 개선
   - 포커스 표시 강화

4. **모바일 경험 향상** (지은, 수진 동의)
   - 다크모드 추가
   - 터치 최적화
   - 키보드 UI 대응

## ⚖️ 페르소나 간 논쟁점

### 논쟁 1: 채팅 인터페이스 복잡도

**👨‍💻 민수:** "탭이 많아도 좋아요. 빠르게 전환할 수 있으니까."

## 🎨 시각적 디자인 평가

### 우수한 점

1. **색상 팔레트:** 청록색-보라색 그라데이션이 현대적이고 의료 서비스에 적합
2. **여백 사용:** 적절한 padding으로 답답하지 않음
3. **카드 디자인:** 섀도우와 라운드 코너가 깔끔함
4. **일관성:** 버튼 스타일, 색상 사용이 일관적

### 개선이 필요한 점

1. **대비 부족:** Placeholder 텍스트가 너무 연함 (WCAG 기준 미달 가능성)
2. **폰트 크기:** 본문 텍스트가 14px로 작음 (16px 권장)
3. **아이콘 크기:** 일부 아이콘이 너무 작아 터치하기 어려움
4. **포커스 표시:** 키보드 포커스 시 outline이 불명확

## 🐛 발견된 버그 및 이슈

### UI 버그
1. **대화 목록 중복:** 동일한 "의료 복지 상담" 대화방이 2개 표시됨
2. **전송 버튼 상태:** 입력창이 비어있을 때 disabled이지만 시각적 차이 미미

### UX 이슈
1. **로딩 표시 없음:** 버튼 클릭 후 응답이 늦으면 사용자가 다시 클릭할 가능성
2. **에러 복구 어려움:** 에러 발생 시 뒤로가기 외 복구 방법 불명확

### 접근성 이슈
1. **색상만으로 구분:** "완료" 배지가 색상으로만 표시 (색맹 사용자 고려 필요)
2. **포커스 트랩 없음:** 모달/사이드바 열렸을 때 ESC나 배경 클릭으로 닫히는지 불명확
