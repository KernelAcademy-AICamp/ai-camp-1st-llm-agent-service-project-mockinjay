## Key Takeaways

1. **Session Flow:**
   - Create once per user visit
   - Persist in MongoDB for reliability
   - Auto-expire after 30 minutes

2. **Streaming Flow:**
   - SSE for real-time updates
   - Chunk responses for smooth UX
   - Save complete conversations to DB

3. **History Flow:**
   - Store every conversation
   - Fast retrieval with indexes
   - Support multi-room organization

4. **Error Handling:**
   - Validate at every step
   - Return user-friendly errors
   - Log for debugging

5. **Performance:**
   - Use indexes on all queries
   - Cache session info in memory
   - Stream responses for perceived speed
